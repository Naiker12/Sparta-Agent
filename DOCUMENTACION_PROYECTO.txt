================================================================================
 1. ¿QUÉ ES SPARTA AGENT?
================================================================================

Sparta Agent es una aplicación de escritorio para orquestar agentes de IA,
construida con React + TypeScript + Vite + Electron. Proporciona una interfaz
de chat para interactuar con múltiples agentes de IA (codificación,
investigación, automatización, gestión de proyectos, orquestación MCP).

Soporta el Protocolo de Contexto de Modelo (MCP) para conectar servidores de
herramientas externas, tiene un sistema multi-tema con 13 temas visuales
(7 oscuros + 6 claros), sistema de i18n español/inglés, administra
conversaciones en sesiones, canales, skills reutilizables, memoria persistente,
proyectos y proveedores de modelos. Es como un "IDE de agentes de IA" local-first.

Actualmente está en etapa temprana (v0.5.0) — muchas características
placeholder, pero con sistemas funcionales de configuración, temas, i18n,
proveedores de modelo y gestión de datos.

 4. ARQUITECTURA
================================================================================

 4.1 Flujo de la Aplicación:

  index.html
    → src/main.tsx (React root)
      → AppShell.tsx (shell raíz)
        → TitleBar.tsx (AppMenu + tabs navegación + botón settings)
        → Sidebar.tsx (navegación por vistas + búsqueda + secciones)
        → Panel Activo (según mainView.type):
            - chat     → ChatArea.tsx
            - editor   → EditorPanel (placeholder)
            - terminal → TerminalPanel (placeholder)
            - agents   → AgentsPanel (placeholder)
            - sessions → SessionsView.tsx
            - skills   → SkillsView.tsx
            - mcp      → McpView.tsx
            - channels → ChannelsView.tsx
            - memory   → MemoryView.tsx
        → StatusBar.tsx (barra de estado inferior)
        → SettingsDialog.tsx (modal, renderizado condicional)

 4.2 Flujo de Estado:

  Zustand Stores ←(subscribe)→ React Components
       ↓
  types/ (interfaces compartidas)
       ↓
  persist middleware → localStorage (9 stores)

 4.3 Flujo de Datos:

  - Zustand stores son la única fuente de verdad
  - 9 stores usan persist middleware (chat, channel, memory, project,
    provider, settings, skill, theme, ui) — guardan en localStorage
  - 3 stores son en memoria (agent, event-bus, mcp)
  - Los hooks son wrappers delgados alrededor de los stores
  - Los componentes leen de stores directamente o mediante hooks
  - Event bus (event-bus.store.ts) es un sistema pub/sub en memoria:
      - Eventos de agente: agent:started, agent:completed, agent:error
      - Eventos de thinking: thinking:started, thinking:token, thinking:completed
      - Eventos de herramientas: tool:called, tool:result, tool:error
      - Eventos MCP: mcp:connected, mcp:tool_discovered, mcp:disconnected
      - Eventos streaming: stream:token, stream:completed, stream:error
      - Eventos pipeline: pipeline:step
      - Eventos chat: chat:message, message:deleted, message:edited, message:shared
      - Eventos sesión: session:created, session:switched
      - Eventos proyecto: project:created, project:switched, project:deleted
      - Eventos skill: skill:created, skill:updated, skill:deleted, skill:invoked
      - Eventos canal: channel:created, channel:message, channel:deleted
      - Eventos memoria: memory:added, memory:updated, memory:deleted

 4.4 Integración Electron:

  - vite-plugin-electron/simple compila:
      electron/main.ts → dist-electron/main.js
      electron/preload.ts → dist-electron/preload.mjs
  - En dev: carga desde Vite dev server URL
  - En prod: carga dist/index.html
  - titleBarStyle: 'hidden' + titleBarOverlay (color #0C0C10, symbolColor #9CA3AF, height 38)
  - contextIsolation: true, nodeIntegration: false
  - IPC handlers: win:minimize, win:maximize, win:close, win:isMaximized
  - Exposición via contextBridge:
      window.electronAPI → min/max/close/isMaximized/onMaximizedChange
      window.electron → IPC genérico (on/send/invoke)
      window.sparta → eventos (onEvent/sendEvent)

 4.5 Routing:

  No hay router. La navegación es por estado de MainView (ui.store.ts):
    MainView = { type: 'chat' | 'editor' | 'terminal' | 'agents'
                 | 'sessions' | 'skills' | 'mcp' | 'channels' | 'memory' }

 8. SISTEMA DE TEMAS (Design Tokens)
================================================================================

  13 Temas: midnight (default), obsidian, emerald, rose, amber, dracula,
            catppuccin, nord (oscuros)
            linen, sage, mist, sand, light (claros)

  Aplicado via atributo data-theme en <html>. Cada tema sobreescribe las
  variables CSS en :root (definidas en globals.css, con archivos individuales
  en src/styles/themes/ como respaldo).

  Además, el atributo data-theme-mode se setea automáticamente al cambiar
  de tema: "dark" para los 8 temas oscuros (midnight, obsidian, emerald,
  rose, amber, dracula, catppuccin, nord) y "light" para los 5 claros.
  Esto se usa para la clase .brand-icon-mono que invierte logos
  monocromáticos (Anthropic, OpenAI) en modos oscuros via filter: invert(1).

  DARK_THEMES se exporta desde theme.store.ts y se usa en el hook useTheme()
  para exponer isDark: boolean a los componentes.
