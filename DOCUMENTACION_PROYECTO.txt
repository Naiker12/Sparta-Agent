================================================================================
 1. ¿QUÉ ES SPARTA AGENT?
================================================================================

Sparta Agent es una aplicación de escritorio para orquestar agentes de IA,
construida con React + TypeScript + Vite + Electron. Proporciona una interfaz
de chat para interactuar con múltiples agentes de IA (codificación,
investigación, automatización, gestión de proyectos, orquestación MCP).

Soporta el Protocolo de Contexto de Modelo (MCP) para conectar servidores de
herramientas externas, tiene un sistema multi-tema con 9 temas visuales, y
administra conversaciones en sesiones. Es como un "IDE de agentes de IA"
local-first.

Actualmente está en etapa temprana (v0.0.0) — muchas características son
placeholders. No tiene backend ni integración real con modelos de IA.

 4. ARQUITECTURA
================================================================================

 4.1 Flujo de la Aplicación:

  index.html
    → src/main.tsx (React root)
      → AppShell.tsx (shell raíz)
        → TitleBar.tsx (navegación por tabs + botón settings)
        → Sidebar.tsx (lista de sesiones)
        → Panel Activo (según activeTab):
            - ChatArea.tsx (chat)
            - EditorPanel (placeholder)
            - TerminalPanel (placeholder)
            - AgentsPanel (placeholder)
        → StatusBar.tsx (barra de estado)
        → SettingsDialog.tsx (modal, renderizado condicional)

 4.2 Flujo de Estado:

  Zustand Stores ←(subscribe)→ React Components
       ↓
  types/ (interfaces compartidas)

 4.3 Flujo de Datos:

  - Zustand stores son la única fuente de verdad
  - Los hooks son wrappers delgados alrededor de los stores
  - Los componentes leen de stores directamente o mediante hooks
  - Event bus (event-bus.store.ts) es un sistema pub/sub en memoria:
      - Eventos de agente: agent:started, agent:completed, agent:error
      - Eventos de thinking: thinking:started, thinking:token, thinking:completed
      - Eventos de herramientas: tool:called, tool:result, tool:error
      - Eventos MCP: mcp:connected, mcp:tool_discovered, mcp:disconnected
      - Eventos streaming: stream:token, stream:completed, stream:error
      - Otros: pipeline:step, chat:message, session:created, session:switched

 4.4 Integración Electron:

  - vite-plugin-electron/simple compila:
      electron/main.ts → dist-electron/main.js
      electron/preload.ts → dist-electron/preload.mjs
  - En dev: carga desde Vite dev server URL
  - En prod: carga dist/index.html
  - titleBarStyle: 'hiddenInset' (titlebar nativa macOS)
  - contextIsolation: true, nodeIntegration: false
  - Exposición via contextBridge: window.electronAPI (min/max/close),
    window.electron (IPC genérico), window.sparta (eventos)

 4.5 Routing:

  No hay router. La navegación es por estado de tab:
    activeTab: 'chat' | 'editor' | 'terminal' | 'agents'

 6. SISTEMA DE ESTADOS (Zustand Stores)
================================================================================

  agent.store.ts
    └── Estado: agents[], activeAgentId, tasks{}
    └── Agentes default: Coding Agent, Research Agent, Automation Agent,
                         Project Agent, MCP Agent
    └── Acciones: registerAgent, setActiveAgent, updateAgentStatus,
                  addTask, updateTask

  chat.store.ts
    └── Estado: sessions[], activeSessionId, messagesBySession{}, isStreaming
    └── Acciones: createSession (retorna UUID), switchSession, deleteSession,
                  addMessage, updateMessage, setStreaming, getActiveMessages

  event-bus.store.ts
    └── Estado: events[] (últimos 500), listeners (Set)
    └── Acciones: dispatch (notifica listeners), subscribe (retorna unsubscribe), clear

  mcp.store.ts
    └── Estado: servers[] (3 servidores default)
    └── Servidores default: Filesystem (conectado, 3 tools), Git (conectado, 3 tools),
                            SQLite (desconectado)
    └── Acciones: addServer, removeServer, setConnected, addTool,
                  removeTool, toggleServer

  settings.store.ts
    └── Estado: activeTab, settingsOpen, defaultModel, input, activeModel,
                memoryEnabled, apiKeys{}
    └── Acciones: setActiveTab, openSettings, closeSettings, setInput,
                  sendMessage, setDefaultModel, setApiKey, toggleMemory

  theme.store.ts
    └── Estado: theme (persistido en localStorage key 'sparta-theme')
    └── Acciones: setTheme (actualiza DOM data-theme + store)
    └── Helper: initTheme() — lee localStorage y aplica tema al boot

  ui.store.ts
    └── Estado: sidebarOpen, sidebarView, contextPanelOpen, mainPanel, activeTab
    └── Acciones: toggleSidebar, setSidebarView, toggleContextPanel,
                  setMainPanel, setActiveTab
