================================================================================
1. CAPTURA DE REASONING — MÚLTIPLES FORMATOS
================================================================================

HERMES AGENT (completo):
  • reasoning — texto plano (DeepSeek, Qwen)
  • reasoning_content — raw del proveedor (OpenAI-compatible)
  • reasoning_details — array objetos (OpenRouter unified format)
  • Bloques type:"thinking" en content (Anthropic, DeepSeek V4 Pro)
  • Tags inline <think>/<thinking>/<reasoning> en content
  • codex_reasoning_items — Codex Responses API
  Prioridad estricta: 1→2→3→4→5, no duplica.

SPARTA AGENT (actual):
  • reasoning_content de delta (OpenAI-compatible) — openai.transport.ts:121
  • Bloques type:"thinking" de Anthropic — anthropic.transport.ts:110-116
  • Tags inline <think>/<thinking>/<reasoning> — think_scrubber.py:3-4
  • reasoning en additional_kwargs — event_bridge.py:140
  ✗ NO maneja reasoning_details de OpenRouter
  ✗ NO maneja codex_reasoning_items
  ✗ NO tiene prioridad ordenada entre formatos

CAMBIOS NECESARIOS:
  [x] Añadir extracción de reasoning_details en openai.transport.ts
      (OpenRouter unified format: delta.reasoning_details array)
  [~] Implementar extracción con prioridad ordenada en Python event_bridge.py
      (parcial: se extrae de metadata, bloques, y tags inline, pero sin prioridad 1→2→3 formal)
  [x] Preservar reasoning_details como campo separado en Message (types/chat.ts)
  [x] Añadir codex_reasoning_items al tipo Message

2. STREAMING THINK SCRUBBER — PARTIAL TAGS
================================================================================

HERMES AGENT (386 líneas, agent/think_scrubber.py):
  • Máquina de estados con _in_block, _buf, _last_emitted_ended_newline
  • Block-boundary rules: open tag solo se reconoce si está al inicio del
    stream, después de newline, o solo whitespace antes
  • Pares cerrados se suprimen SIEMPRE (incluso mid-line)
  • Partial tag detection: retiene tails tipo "<thin" hasta próximo chunk
  • flush() al final suelta texto retenido

SPARTA AGENT (78 líneas, python/sparta_ai/streaming/think_scrubber.py):
  • Implementación básica con feed() y flush()
  • Busca open/close tags pero SIN block-boundary rules
  ✗ NO distingue entre tags en boundary vs mid-sentence
  ✗ Propenso a falsos positivos: "usa <think> en tu código" se suprime
  ✗ Manejo de partial tags muy simple

CAMBIOS NECESARIOS:
  [x] Mejorar StreamingThinkScrubber con block-boundary rules
  [x] Añadir detección de partial tags con buffer tail
  [x] Implementar regla: pares cerrados se suprimen SIEMPRE,
      opens solo en boundary de línea
  [x] Tests existentes pasan (7 tests de think_scrubber + 7 de streaming)
  [~] Tests adicionales para boundary cases mejorados (pendiente: casos mid-sentence con prosa)

3. COLLAPSE/EXPAND — UX Y ESTADO
================================================================================

HERMES AGENT (ThinkingDisclosure + ReasoningAccordionGroup):
  • Auto-open mientras streaming (pending=true)
  • Auto-close cuando completed
  • User override: si el usuario hace clic, su elección PERSISTE
  • Shift+click expande todas las secciones
  • reasoning.delta vs thinking.delta separados
  • ReasoningAccordionGroup agrupa reasoning parts consecutivos
  • Self-gating: no renderiza si no hay contenido (evita bloques vacíos)
  • StreamStallIndicator: detecta silencio >2s y muestra indicador

SPARTA AGENT (TimelineBlock.tsx):
  • Auto-open durante streaming
  • Auto-close tras 600ms al completar
  • User override: trackea con userToggled ref pero NO persiste
    entre mensajes (se pierde en cada nuevo mensaje)
  • canToggle se deshabilita durante streaming (no se puede colapsar
    mientras stremea — UX pobre)
  ✗ NO persiste preferencia de collapse/expand entre sesiones
  ✗ NO agrupa reasoning parts consecutivos
  ✗ NO tiene self-gating para bloques vacíos
  ✗ No hay indicador de stall (stream congelado)

CAMBIOS NECESARIOS:
  [x] Persistir estado de collapse en localStorage por messageId
  [x] Permitir toggle incluso durante streaming (se eliminó canToggle)
  [x] Añadir self-gating: no renderiza si content vacío y status===completed
  [ ] Agrupar reasoning parts consecutivos en un solo disclosure (ReasoningAccordionGroup)
  [ ] Implementar StreamStallIndicator (timer 2s de silencio)
  [ ] Añadir preview mode (max-h-40 con fade CSS)
  [ ] Shift+click expande todas las secciones

  ✗ NO hay lógica para evitar doble callback

CAMBIOS NECESARIOS:

    y puede romper la alternancia de roles

  [ ] ⚠️ Añadir merge de user messages adyacentes

13. MEJORES PRÁCTICAS DE HERMES APLICABLES
================================================================================

1. Prompt caching: reasoning como campo separado del content → NO invalida cache
2. No duplicación: si ya se mostró en streaming, no repetir en mensaje final
3. Filtrado honesto: dropear thinking-only turns en COPIA, no en histórico
4. Persistencia fiel: guardar TODOS los formatos, no solo texto plano
5. Block-boundary rules: evitar falsos positivos con tags mencionados en prosa
6. Partial tag handling: retener tails entre chunks
7. User override: si el usuario hace clic, su estado persiste
8. Self-gating: no renderizar bloques vacíos
9. Cross-provider safety: limpiar/inyectar reasoning según vendor activo
10. Reasoning effort granular: none → xhigh, mapeado por vendor
