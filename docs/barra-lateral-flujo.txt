7.  CICLO DE VIDA DEL SIDEBAR
═══════════════════════════════════════════════════════════════

   Montaje inicial:
   ──────────────
   1. AppShell llama a initTheme() → aplica data-theme en <html>
   2. TitleBar se renderiza con botón toggle
   3. SidebarProvider se monta con open=ui.store.sidebarOpen
   4. SidebarProvider registra listener Ctrl+B (shadcn interno)
   5. AppSidebar lee todas las stores y renderiza

   Interacción usuario:
   ────────────────────
   Click "Nueva sesión"
     → chatStore.createSession() → nueva Session en store
     → uiStore.setMainView({ type: 'chat' }) → cambia panel
     → AppSidebar se re-renderiza (nueva sesión aparece en Recientes)

   Click nav item (ej. Skills)
     → uiStore.setMainView({ type: 'skills' })
     → Panel cambia a <SkillsView>

   Click Ctrl+B
     → SidebarProvider.onKeyDown → toggleSidebar()
     → Llama onOpenChange → uiStore.setState({ sidebarOpen })
     → Sidebar se colapsa/expande usando CSS transitions

   Click SessionItem
     → chatStore.switchSession(id)
     → SidebarMenuButton.isActive = true
     → UIStore.setMainView({ type: 'chat', sessionId: id })

   Click ⋮ → Fijar
     → chatStore.pinSession(id)
     → SessionItem se mueve de Recientes → Fijados

   Click ⋮ → Eliminar
     → ConfirmDeleteDialog → chatStore.deleteSession(id)
     → Sesión desaparece

   Click ⋮ → Renombrar
     → prompt() → chatStore.renameSession(id, newTitle)

   Búsqueda:
     → query state cambia
     → filtered = sessions.filter(title.includes(query))
     → pinned/unpinned se recalculan
     → Solo sesiones que matchean aparecen

9.  SIDEBAR EN ESTADO COLLAPSED (icon)
═══════════════════════════════════════════════════════════════

   Cuando collapsible="icon" y el sidebar está cerrado:
   ─────────────────────────────────────────────────────────
   • Ancho: 3rem (48px) — definido por --sidebar-width-icon
   • Solo iconos visibles en los SidebarMenuButton
   • Texto oculto con clase: group-data-[collapsible=icon]:hidden
   • Tooltips aparecen en cada SidebarMenuButton (prop tooltip)
   • SidebarHeader se oculta (group-data-[collapsible=icon]:hidden)
   • SidebarFooter igual
   • SidebarRail visible para expandir
   • SidebarMenuBadge oculto
   • SidebarGroupLabel oculto
   • SidebarMenuAction.showOnHover → opacity-0 (no hay hover real)

13. NOTAS TÉCNICAS
═══════════════════════════════════════════════════════════════

   • asChild no disponible: Base UI no tiene el patrón asChild
     de Radix. Todo componente trigger (DropdownMenuTrigger,
     TooltipTrigger) renderiza su propio <button> y no acepta
     que se le pase un child como reemplazo. En su lugar se
     pasan className y onClick directamente al trigger.

   • CVA en SidebarMenuButton: usa class-variance-authority
     con variantes: variant (default/outline), size (default/sm/lg).

   • group-data-[collapsible=icon]: El sidebar usa el data-
     attribute data-collapsible="icon" para ocultar/mostrar
     elementos según el estado colapsado. Los grupos CSS
     (group/sidebar-wrapper, group/menu-item) permiten
     selectores descendientes.

   • Ctrl+B: manejado internamente por SidebarProvider.
     No necesita código adicional en AppShell.

   • Persistencia: El estado de sidebar open/close se guarda
     en cookie (sidebar_state) por shadcn, y en localStorage
     por Zustand (sparta-ui). Ambos se sincronizan vía el
     onOpenChange del SidebarProvider.

   • Tailwind v4: No hay tailwind.config.js. La configuración
     es CSS-first con @import "tailwindcss" y @theme inline.
     Los valores arbitrarios como text-[13px] o transition-[width]
     son totalmente soportados.
