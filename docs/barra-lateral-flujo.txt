1.  RESUMEN / ARQUITECTURA GENERAL
═══════════════════════════════════════════════════════════════

La barra lateral fue reconstruida desde cero. No es el sidebar
estándar de shadcn/ui tal cual; usamos un componente base propio
(src/components/ui/sidebar.tsx) que conserva la API de shadcn pero
está implementado con primitivas de @base-ui/react (merge-props,
use-render, dialog, etc.).

Sobre ese componente base, AppSidebar.tsx dibuja una UI propia:
  • Logo SPARTA AGENT
  • Botón “Nueva sesión”
  • 5 ítems de navegación principales
  • Buscador de sesiones
  • Secciones “Fijados” y “Recientes”
  • Footer con Configuración

La navegación de vistas Chat/Editor/Terminal/Agents ya NO vive en
el sidebar; ahora está en la TitleBar.

8.  CICLO DE VIDA E INTERACCIONES
═══════════════════════════════════════════════════════════════

   Montaje inicial:
   ──────────────
   1. AppShell llama a initTheme() → aplica data-theme en <html>
   2. TitleBar se renderiza con botón toggle del sidebar
   3. SidebarProvider se monta con open=ui.store.sidebarOpen
   4. SidebarProvider registra atajo Ctrl+B internamente
   5. AppSidebar lee stores y renderiza

   Interacción usuario:
   ────────────────────
   Click “Nueva sesión"
     → chatStore.createSession()
     → uiStore.setMainView({ type: 'chat' })
     → AppSidebar se re-renderiza

   Click nav item (ej. Skills)
     → uiStore.setMainView({ type: 'skills' })
     → Panel cambia a <SkillsView>

   Click toggle en TitleBar (o Ctrl+B)
     → uiStore.toggleSidebar()
     → SidebarProvider recibe nuevo open
     → Sidebar se colapsa/expande con transición CSS

   Redimensionar sidebar
     → SidebarResizeHandle.onMouseMove → setSidebarWidth(clientX)
     → ui.store guarda entre 220px y 480px
     → SidebarProvider recibe nuevo --sidebar-width

   Click SessionItem
     → chatStore.switchSession(id)
     → uiStore.setMainView({ type: 'chat', sessionId: id })

   Menú ⋮ → Fijar / Quitar fijado
     → chatStore.pinSession(id)
     → Sesión se mueve entre Fijados y Recientes

   Menú ⋮ → Renombrar
     → prompt() → chatStore.renameSession(id, newTitle)

   Menú ⋮ → Compartir
     → Dialog con enlace sparta://session/{id} + Copiar

   Menú ⋮ → Eliminar
     → ConfirmDeleteDialog → chatStore.deleteSession(id)

   Búsqueda:
     → query state local
     → sessions.filter(title.toLowerCase().includes(query))
     → pinned/unpinned se recalculan

9.  ESTADO COLLAPSED Y RESIZE
═══════════════════════════════════════════════════════════════

   Colapsado (collapsible="icon"):
   ──────────────────────────────
   • Ancho base: 280px  (--sidebar-width desde ui.store)
   • Ancho collapsed: 48px (--sidebar-width-icon = 3rem)
   • Toggle controlado por uiStore.toggleSidebar()
   • Atajo Ctrl+B registrado en SidebarProvider
   • data-state="collapsed" se setea en el wrapper
     <div data-slot="sidebar"> (ancestro de sidebar-inner)
   • Estilos en globals.css ocultan texto/badges/separadores y
     centran los iconos en modo collapsed usando selectores
     descendientes [data-state="collapsed"] .clase.
   • Clases añadidas en AppSidebar para soportar el bloque:
     .btn-label (texto "Nueva sesión"), .footer-label ("Configuración"),
     .sidebar-new-session-wrapper y .sidebar-footer-wrapper.
   • En collapsed se reduce el padding de wrappers y logo, se achica
     la imagen del logo a 28px y los botones usan width: 100% con
     padding horizontal reducido para no desbordarse del ancho de 48px.

   Resize:
   ───────
   • Componente: SidebarResizeHandle.tsx
   • Límites: 220px mínimo, 480px máximo
   • Persistencia: ui.store.sidebarWidth se guarda en localStorage
   • Cursor: col-resize mientras se arrastra
   • Visual: línea de 2px derecha del sidebar que resalta en --accent

13. NOTAS TÉCNICAS / GOTTCHAS
═══════════════════════════════════════════════════════════════

   • Sidebar base propio: src/components/ui/sidebar.tsx es un
     fork/custom que reemplaza las primitivas Radix por Base UI.
     Aunque exporta la API clásica de shadcn, AppSidebar no usa
     la mayoría de esos exports (no usa SidebarMenuButton, etc.).

   • asChild no disponible: Base UI no tiene el patrón asChild de
     Radix. Los triggers de dropdown/tooltip renderizan su propio
     <button>; se pasan className y onClick directamente.

   • Toggle del sidebar: TitleBar ya no usa SidebarTrigger de
     shadcn; llama directamente a uiStore.toggleSidebar().

   • Navegación principal: los tabs Chat/Editor/Terminal/Agents
     están en TitleBar, no en el sidebar.

   • Ancho del sidebar: se controla vía CSS custom property
     --sidebar-width inyectada por AppShell, actualizada por
     SidebarResizeHandle y persistida en ui.store.

   • Modo collapsed: la lógica de colapso funciona (ancho 48px) y
     los estilos custom ya ocultan textos, centran iconos y ocultan
     el indicador activo vía [data-state="collapsed"] en globals.css.

   • Persistencia: el estado open/close y el ancho se guardan en
     localStorage bajo la key "sparta-ui". El provider base también
     escribe una cookie "sidebar_state", pero el source of truth
     para la app es Zustand.

   • Tailwind v4: no hay tailwind.config.js; todo es CSS-first con
     @import "tailwindcss" y @theme inline.
