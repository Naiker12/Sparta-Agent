P: ¿Cómo manejas el historial de conversación cuando se vuelve muy largo?
R: Via context_manager.py:16-53. Se comprimen mensajes antiguos cuando:
   - Hay más de 20 mensajes (MAX_MESSAGES_BEFORE_COMPRESS = 20)
   - Y más de 40,000 caracteres totales (MAX_CHARS_BEFORE_COMPRESS = 40_000)

   Cuando se activa, se preserva:
   - El primer mensaje system (si existe)
   - Los últimos 6 mensajes
   - Un resumen generado por el LLM de todo lo intermedio

P: ¿Existe algún proceso de compresión, resumen o truncamiento?
   ¿Invoca este proceso al LLM en medio del flujo principal?
R: SÍ, invoca al LLM directamente en el flujo principal:

   context_manager.py:56-70:
   async def _summarize(messages: list[dict], llm: Any) -> str:
       summary_prompt = (
           "Resume la siguiente conversación en 3-5 puntos clave...\n\n"
           + "\n".join(f"{m['role'].upper()}: ...")
       )
       response = await llm.ainvoke([{"role": "user", "content": summary_prompt}])
       return str(response.content)

   Esto bloquea la preparación del agente hasta que el LLM termine de
   generar el resumen. Con un modelo local, esto puede tardar 2-10 segundos.
   Se ejecuta DENTRO de prepare_agent() antes de construir el grafo.

   NOTA: Ahora está paralelizado con las demás tareas via asyncio.gather()
   (Algoritmo B aplicado), pero sigue bloqueando porque compress_if_needed
   necesita el LLM como argumento.

3. PIPELINE DE STREAMING Y TRANSPORTE
================================================================================

P: ¿Cómo viajan los tokens desde el backend de Python al frontend en React?
R: DOS MODOS según la plataforma:

   MODO ELECTRON (IPC via stdin/stdout):
   Python sidecar → sys.stdout.write(JSON + "\n") → stdout
   → Electron main process (sidecar.ipc.ts:117-137) lee stdout
   → JSON.parse por línea → EventEmitter
   → chat/index.ts → webContents.send('sparta:event')
   → React: window.sparta.onEvent(handler) → _handleEvent()

   MODO WEB (WebSocket):
   Python sidecar → FastAPI WebSocket (server_web.py:93)
   → ws.send_text(JSON) → WebSocket
   → React: WebAdapter.onmessage → JSON.parse → handlers.forEach(h => h(event))

   CÓDIGO DE EMISIÓN (event_bridge.py:47-69):
   def _emit(request_id, event, data, stream_state):
       msg = {"id": request_id, "event": event}
       if data: msg["data"] = data
       sys.stdout.write(json.dumps(msg, ensure_ascii=False) + "\n")
       # Flush cada _FLUSH_BATCH_SIZE tokens (ahora = 1)
       if should_flush:
           sys.stdout.flush()

P: ¿Existe algún buffer o acumulador de tokens intermedio?
R: SÍ, en DOS capas:

   PYTHON (event_bridge.py:24):
   _FLUSH_BATCH_SIZE = 1  (antes era 8)
   Se hace flush a stdout después de cada token de streaming.
   Los eventos de control (tool:called, stream:completed, etc.)
   siempre se flushen inmediatamente.

   FRONTEND (useStreamEvents.ts:55-109):
   RAF-throttled flush usando requestAnimationFrame:
   - _queueContent() bufferiza tokens en _writeBuf
   - _queueThinking() bufferiza reasoning en _thinkBuf
   - _scheduleFlush() usa requestAnimationFrame (~16ms = 60fps)
   - _flushBoth() vacía ambos buffers al store Zustand
   Esto evita re-render storms sin agregar latencia perceptible.

P: ¿Qué componentes procesan el stream de salida en tiempo real?
R: Cada token de LM Studio pasa por estos filtros EN SERIE:

   1. _extract_reasoning_content(chunk)
      Busca reasoning_content en metadata del chunk (DeepSeek/OpenRouter)
      → chat_model_stream.py:52

   2. StreamingThinkScrubber.feed(content)
      Detecta y separa tags <think>/<thinking>/<reasoning>
      Bloques closed-pair mid-line: siempre suprimidos
      Bloques open-tag: reconocidos solo al inicio de línea
      → think_scrubber.py:37-143

   3. RepetitionGuard.feed(visible_text)
      Detecta loops de n-gram (12-char window, 6 repeticiones)
      Detecta loops de frase (40-char phrase, 3 repeticiones)
      Skip dentro de bloques de código (```)
      → repetition_guard.py:58-83

   4. _detect_and_emit_skill(thinking_text)
      Regex contra patrones de skills activos
      Emite skill:activated si matchea
      → chat_model_stream.py:19-28

   5. Skip-mode logic (deduplicación de texto repetido)
      Compara emitted_text con pending para evitar duplicación
      → chat_model_stream.py:107-133

P: ¿Cómo está estructurado el grafo de tu agente?
R: Grafo LangGraph StateGraph con 4 nodos principales:

   Estructura (sparta_agent.py:564-593):

   ┌─────────┐
   │  START   │
   └────┬────┘
        ▼
   ┌─────────┐
   │  agent   │ ◄──────────────────────────┐
   └────┬────┘                              │
        │                                   │
        ▼ should_continue()                 │
   ┌────┴────────────────────┐              │
   │                         │              │
   ▼                         ▼              │
   ┌───────────┐    ┌──────────────────┐    │
   │   tools   │    │ subagent_coord   │    │
   └─────┬─────┘    └────────┬─────────┘    │
         │                   │              │
         ▼ should_reflect()  │              │
   ┌─────┴──────┐            │              │
   │            │            │              │
   ▼            ▼            │              │
   ┌────────────┐  ┌─────┐  │              │
   │ reflection  │  │ END │  │              │
   └──────┬─────┘  └─────┘  │              │
          │                  │              │
          └──────────────────┴──────────────┘

   NODOS:
   - agent: agent_node() - LLM invoke con herramientas bindeadas
   - tools: tool_node() - Ejecuta LangGraph ToolNode con herramientas
   - subagent_coordinator: subagent_node() - Lanza subgrafos en paralelo
   - reflection: reflection_node() - Analiza errores de tools, reintentos

   SUBAGENTES (sparta_agent.py:423-555):
   - delegate_research → build_research_graph() (búsqueda web profunda)
   - delegate_code     → build_code_graph() (ejecución de código)
   - delegate_memory   → build_memory_graph() (recuperación semántica)
   - delegate_review   → build_review_graph() (revisión de cambios)
   Timeout de subagentes: 120 segundos (asyncio.wait_for)
   Ejecución: paralela via asyncio.gather

   CONDICIONES DE ROUTING:
   - should_continue(): tool_calls → "tools"; delegate_* → "subagent";
     loop detectado o tool_limit → "agent" (forced summary); sin tools → "__end__"
   - should_reflect(): si tools fallaron → "reflection"; si no → "agent"
   - MAX_TOOL_CALLS_PER_TURN = 8 (hard limit)

P: ¿Cómo se bindean las herramientas al LLM?
R: Tres variantes de LLM con diferentes niveles de herramientas
   (sparta_agent.py:81-86):

   LLM_BUILD (modo agente completo):
     llm_with_tools = llm.bind_tools(all_tools + delegate_tools)
     all_tools = 15+ herramientas (read/write/patch/delete/terminal/mcp/skills...)
     delegate_tools = [research_topic, execute_code_task, recall_memories, review_changes]

   LLM_PLAN (modo planificación read-only):
     llm_plan = llm.bind_tools(plan_tools + delegate_tools)
     plan_tools = PermissionPolicy("plan").filter_tools(all_tools) → solo lectura

   LLM_CHAT (modo conversacional):
     llm_chat = llm.bind_tools(chat_tools + [research_topic])
     chat_tools = PermissionPolicy("chat").filter_tools(all_tools) → lectura+busqueda+memoria

   Selección en agent_node():
     if is_forced_summary → llm.ainvoke (sin tools)
     elif scope=="readonly" or policy_mode=="plan" → llm_plan
     elif scope=="chat" → llm_chat
     else → llm_with_tools

--- chat_model_stream.py (handler de streaming de tokens) ---

Ubicación: python/sparta_ai/streaming/handlers/chat_model_stream.py

async def handle_chat_model_stream(
    emit_fn, emit_control_fn, event, data, name,
    stream_state, request_id, base_payload
) -> bool:
    chunk = data.get("chunk")
    if chunk is None:
        return False

    # 1. Buscar reasoning nativo en metadata
    reasoning_content = _extract_reasoning_content(chunk)
    if reasoning_content:
        if not stream_state["thinking_active"]:
            emit_control_fn(*reasoning_events.thinking_started(base_payload))
            stream_state["thinking_active"] = True
        stream_state["reasoning_tokens"] += len(reasoning_content.split())
        emit_fn(*reasoning_events.thinking_token(base_payload, reasoning_content))
        _detect_and_emit_skill(emit_control_fn, reasoning_content, stream_state, base_payload)
        return False

    content = getattr(chunk, "content", "")

    # 2. Content como lista de bloques (Anthropic-style)
    if isinstance(content, list):
        for block in content:
            if _is_reasoning_block(block):    # Bloque think nativo
                # ... emit thinking:token
            elif _is_text_block(block):       # Bloque de texto visible
                text = _block_text(block)
                rep_guard = stream_state.setdefault("_rep_guard", RepetitionGuard())
                if rep_guard.feed(text):      # Detectar repetición
                    emit_control_fn("stream:degenerate", {**base_payload})
                    return True               # Abortar stream
                emit_fn("stream:token", {**base_payload, "token": text})

    # 3. Content como string (OpenAI-style + emulated thinking)
    elif isinstance(content, str) and content:
        scrubber = stream_state.setdefault("_scrubber", StreamingThinkScrubber())
        visible, reasoning = scrubber.feed(content)  # Separar think tags
        if reasoning:
            emit_fn(*reasoning_events.thinking_token(base_payload, reasoning))
        if visible:
            rep_guard = stream_state.setdefault("_rep_guard", RepetitionGuard())
            if rep_guard.feed(visible):
                return True  # Abortar por repetición
            emit_fn("stream:token", {**base_payload, "token": visible})
    return False

def single_pass_cleanup(messages: list[dict], vendor: str) -> list[dict]:
    vendor_lower = vendor.lower()
    requires = vendor_lower in _REQUIRE_REASONING_CONTENT   # deepseek, kimi
    rejects = vendor_lower in _REJECT_REASONING_CONTENT     # mistral, groq
    is_anthropic = vendor_lower in _BLOCK_REASONING_VENDORS

    result = []
    for msg in messages:
        m = dict(msg)  # ← Shallow copy (no deepcopy)

        if role == "assistant" and _is_thinking_only(m):
            continue  # Drop thinking-only

        if role == "assistant":
            if requires:    _ensure_reasoning_content(m)
            elif rejects:   _remove_reasoning_fields(m)
            elif is_anthropic:  # Convert to Anthropic blocks
                reasoning = m.get("reasoning_content") or m.get("reasoning")
                if reasoning:
                    m["content"] = [{"type":"thinking","thinking":str(reasoning)},
                                    {"type":"text","text":str(m.get("content",""))}]
                    _remove_reasoning_fields(m)
            else:
                _promote_reasoning_if_present(m)

        result.append(m)

    # Merge user messages adyacentes
    merged = []
    i = 0
    while i < len(result):
        if result[i].get("role") == "user":
            combined = result[i].get("content", "")
            j = i + 1
            while j < len(result) and result[j].get("role") == "user":
                combined += "\n\n" + result[j].get("content", "")
                j += 1
            merged.append({"role": "user", "content": combined})
            i = j
        else:
            merged.append(result[i])
            i += 1
    return merged
